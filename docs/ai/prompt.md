## 1. Prompt 概述

### 1.1 什么是 Prompt

**Prompt（提示词）**：用户输入给 AI 的指令或问题，用于引导 AI 生成期望的输出。

```
Prompt 质量决定 AI 输出质量

❌ 差 Prompt："写点东西"
✅ 好 Prompt："请以 Java 高级开发工程师的身份，写一篇关于 SpringBoot 自动配置原理的技术博客，要求包含源码分析和实战示例，字数 2000 字左右"
```

### 1.2 为什么学 Prompt

```
✅ 提升 AI 输出质量
✅ 减少试错成本
✅ 提高工作效率
✅ 释放 AI 潜力
```

### 1.3 Prompt 工程

**Prompt Engineering**：设计和优化 Prompt 的技术，让 AI 更好地理解意图并生成高质量输出。

## 2. Prompt 基础

### 2.1 Prompt 结构

```
完整 Prompt = 角色 + 任务 + 上下文 + 约束 + 示例

角色：你是什么身份
任务：要做什么
上下文：背景信息
约束：要求和限制
示例：参考样例
```

### 2.2 基础示例

```
❌ 差 Prompt：
"帮我写个总结"

✅ 好 Prompt：
"你是一名资深技术编辑（角色），请总结以下技术文章的核心内容（任务），文章主要讲解 SpringBoot 自动配置原理（上下文），要求 300 字以内，用通俗易懂的语言（约束）"
```

### 2.3 清晰指令

```
✅ 清晰：
"请用 Java 写一个快速排序算法，包含详细注释"

❌ 模糊：
"写个排序"

✅ 清晰：
"请分析这段代码的性能问题，并给出优化建议"

❌ 模糊：
"看看这个代码"
```

## 3. Prompt 技巧

### 3.1 角色设定（Role）

```
作用：让 AI 进入特定角色，输出更专业

示例：
"你是一名资深 Java 架构师，有 10 年微服务架构设计经验..."
"你是一名专业翻译，精通中英文技术文档翻译..."
"你是一名产品经理，擅长需求分析和原型设计..."
"你是一名面试官，正在面试 Java 高级开发工程师..."
```

### 3.2 任务分解（Decomposition）

```
作用：复杂任务拆分为简单步骤

示例：
"请完成以下任务：
1. 分析这段代码的功能
2. 指出潜在的问题
3. 给出优化建议
4. 提供优化后的代码"
```

### 3.3 示例引导（Few-Shot）

```
作用：提供示例，让 AI 理解期望格式

示例：
"请将以下中文翻译成英文：

示例 1：
输入：你好，世界
输出：Hello, World

示例 2：
输入：欢迎使用我们的系统
输出：Welcome to our system

现在翻译：
输入：这是一个测试
输出："
```

### 3.4 思维链（Chain of Thought）

```
作用：引导 AI 逐步思考，提升准确性

示例：
"请解答这道数学题，并展示详细的解题步骤：

题目：一个水池，单独开进水管 6 小时可以注满，单独开排水管 8 小时可以排空。如果同时打开进水管和排水管，多少小时可以注满水池？

请逐步思考：
1. 先计算进水管每小时的注水效率
2. 再计算排水管每小时的排水效率
3. 计算同时开启时的净效率
4. 最后计算注满水池所需时间"
```

### 3.5 约束条件（Constraints）

```
作用：限制输出范围，避免跑题

示例：
"请用 100 字以内总结..."
"请只输出代码，不要解释..."
"请用表格形式展示..."
"请列出 Top 5，并说明理由..."
"请避免使用专业术语，用通俗易懂的语言..."
```

### 3.6 格式指定（Format）

```
作用：指定期望输出格式

示例：
"请用 JSON 格式输出..."
"请用 Markdown 表格展示..."
"请用以下格式：
- 问题：
- 原因：
- 解决方案："
```

## 4. 高级 Prompt

### 4.1 自我一致性（Self-Consistency）

```
方法：让 AI 多次回答，选择最一致的答案

Prompt：
"请解答这个问题，然后重新检查你的答案是否正确：

问题：...
答案：...

现在请重新思考，你的答案是否正确？如果有误，请纠正。"
```

### 4.2 反思 Prompt（Reflection）

```
方法：让 AI 自我评估和改进

Prompt：
"请完成以下任务，然后评估你的输出质量：

任务：写一篇关于 JVM 内存模型的技术文章

完成后请回答：
1. 文章是否涵盖了所有关键点？
2. 是否有错误或遗漏？
3. 如何改进？"
```

### 4.3 对比学习（Comparison）

```
方法：让 AI 对比多个选项

Prompt：
"请对比以下两种方案：

方案 1：使用 Redis 缓存热点数据
方案 2：使用本地缓存（Caffeine）

请从以下维度对比：
1. 性能
2. 可靠性
3. 适用场景
4. 优缺点

最后给出推荐方案。"
```

### 4.4 迭代优化（Iterative）

```
方法：多轮对话逐步优化

第一轮：
"请帮我写一个用户注册接口"

第二轮：
"请添加参数校验功能"

第三轮：
"请添加异常处理和日志记录"

第四轮：
"请补充单元测试"
```

## 5. 实战场景

### 5.1 代码生成

```
Prompt：
"你是一名资深 Java 开发工程师，请帮我写一个用户服务类，要求：

1. 使用 SpringBoot 框架
2. 包含 CRUD 功能
3. 使用 MyBatis-Plus
4. 添加参数校验
5. 添加异常处理
6. 添加详细注释

请提供完整的代码，包括 Entity、Mapper、Service、Controller。"
```

### 5.2 代码解释

```
Prompt：
"请解释以下代码的功能和工作原理：

```java
[粘贴代码]
```

请说明：
1. 这段代码实现什么功能
2. 核心逻辑是什么
3. 使用了哪些技术
4. 有什么潜在问题"
```

### 5.3 Bug 修复

```
Prompt：
"以下代码运行时报错，请帮我找出问题并修复：

```java
[粘贴代码]
```

错误信息：
[粘贴错误日志]

请分析：
1. 错误原因
2. 如何修复
3. 如何避免类似问题"
```

### 5.4 文档写作

```
Prompt：
"你是一名技术文档工程师，请帮我写一份 API 接口文档，要求：

1. 接口名称：用户登录
2. 请求方式：POST
3. 请求参数：username、password
4. 返回参数：token、userInfo
5. 包含请求示例和响应示例
6. 使用 Markdown 格式"
```

### 5.5 数据分析

```
Prompt：
"你是一名数据分析师，请分析以下销售数据：

[粘贴数据]

请回答：
1. 本月总销售额是多少
2. 哪个产品销量最高
3. 销售趋势如何
4. 有什么建议"
```

### 5.6 方案设计

```
Prompt：
"你是一名系统架构师，请设计一个秒杀系统，要求：

1. 支持 10 万 QPS
2. 防止超卖
3. 防止重复购买
4. 高可用

请提供：
1. 系统架构图
2. 核心流程
3. 关键技术选型
4. 优化措施"
```

## 6. 最佳实践

### 6.1 Prompt 模板

```
# 角色设定
你是一名 [角色]，擅长 [技能]

# 任务描述
请 [具体任务]

# 背景信息
[相关上下文]

# 约束条件
- 要求 1
- 要求 2
- 要求 3

# 输出格式
[期望格式]

# 示例
[参考示例]
```

### 6.2 避免的陷阱

```
❌ 避免模糊：
"写点东西" → "写一篇关于 JVM 内存模型的技术文章"

❌ 避免复杂：
一次问多个问题 → 拆分为多个 Prompt

❌ 避免矛盾：
"简短但详细" → 选择其一

❌ 避免假设：
明确说明前提条件
```

### 6.3 优化技巧

```
1. 具体化：越具体越好
2. 结构化：使用分点、编号
3. 示例化：提供参考示例
4. 迭代化：多轮对话优化
5. 验证化：让 AI 自我检查
```

### 6.4 常用句式

```
"请逐步思考..."
"请从以下角度分析..."
"请对比...和..."
"请用...格式输出..."
"请避免..."
"请确保..."
"如果...则..."
```
