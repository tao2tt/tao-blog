## 1. Prompt 简介

### 1.1 什么是 Prompt

**Prompt（提示词）** 是用户输入给大模型的文本，用于指导模型生成期望的输出。

**核心思想：** 好的 Prompt = 好的结果

```
Prompt 质量对比：

❌ 差 Prompt：
"写点什么"
→ 输出：模糊、无针对性

✅ 好 Prompt：
"请用 500 字介绍人工智能的发展历程，包括重要里程碑和代表人物"
→ 输出：结构化、信息丰富、符合预期
```

### 1.2 Prompt 的重要性

| 原因 | 说明 |
|------|------|
| **决定输出质量** | Prompt 越清晰，输出越准确 |
| **降低使用成本** | 好的 Prompt 减少反复调试 |
| **提高效率** | 一次得到满意结果 |
| **解锁高级能力** | 特定 Prompt 激发模型潜能 |

### 1.3 Prompt 工程

**Prompt 工程（Prompt Engineering）** 是设计和优化 Prompt 的艺术和科学。

**核心技能：**
- ✅ 理解模型能力
- ✅ 清晰表达需求
- ✅ 结构化设计
- ✅ 迭代优化

## 2. Prompt 基础结构

### 2.1 基本组成

```
完整 Prompt = 角色 + 任务 + 上下文 + 约束 + 示例
```

| 组成部分 | 说明 | 示例 |
|---------|------|------|
| **角色** | 设定 AI 的身份 | "你是一位资深程序员" |
| **任务** | 明确要做什么 | "请帮我写一个排序函数" |
| **上下文** | 提供背景信息 | "这是一个电商项目的订单模块" |
| **约束** | 限制输出范围 | "代码不超过 50 行，使用 Python" |
| **示例** | 给出参考样例 | "类似这样的格式：..." |

### 2.2 完整示例

```python
# ❌ 简单 Prompt（效果一般）
prompt = "写一个排序函数"

# ✅ 完整 Prompt（效果好）
prompt = """
你是一位资深 Python 程序员（角色）。

请帮我写一个快速排序函数（任务）。
这个函数将用于电商项目的订单排序，数据量较大（上下文）。

要求：
1. 使用 Python 实现
2. 代码不超过 50 行
3. 添加详细注释
4. 包含使用示例（约束）

参考格式：
def quick_sort(arr):
    # 实现逻辑
    return sorted_arr（示例）
"""
```

## 3. 核心技巧

### 3.1 明确具体

```python
# ❌ 模糊
"介绍一下人工智能"

# ✅ 具体
"请用 300 字介绍人工智能的定义和发展历程，包括 3 个重要里程碑"
```

**技巧：**
- ✅ 指定字数
- ✅ 指定格式
- ✅ 指定内容要点

### 3.2 提供示例（Few-shot）

```python
# 零样本（Zero-shot）
prompt = "将以下中文翻译成英文：你好"

# 少样本（Few-shot）- 效果更好
prompt = """
将中文翻译成英文：

中文：早上好
英文：Good morning

中文：谢谢
英文：Thank you

中文：再见
英文：

（模型会自动补全：Goodbye）
"""
```

### 3.3 分步思考（Chain of Thought）

```python
# ❌ 直接问答案
prompt = "小明有 5 个苹果，给了小红 2 个，又买了 3 个，现在有几个？"

# ✅ 引导逐步思考
prompt = """
小明有 5 个苹果，给了小红 2 个，又买了 3 个，现在有几个？

请逐步思考：
1. 初始有几个苹果？
2. 给了小红后剩几个？
3. 又买了几个？
4. 最后总共几个？

请写出计算过程。
"""
```

**效果对比：**

| 方式 | 准确率 |
|------|--------|
| 直接回答 | 60% |
| 逐步思考 | 90%+ |

### 3.4 设定角色

```python
# ❌ 无角色
"这个代码有什么问题？"

# ✅ 设定角色
"""
你是一位资深代码审查专家，有 10 年 Python 开发经验。

请审查以下代码，指出：
1. 潜在 bug
2. 性能问题
3. 代码规范问题
4. 改进建议

代码：
[粘贴代码]
"""
```

### 3.5 格式化输出

```python
# 指定输出格式
prompt = """
请分析这个产品的优缺点，按以下格式输出：

## 优点
1. ...
2. ...

## 缺点
1. ...
2. ...

## 建议
...
"""
```

## 4. 常用场景

### 4.1 内容创作

```python
# 写文章
prompt = """
请写一篇关于"人工智能对未来工作的影响"的博客文章。

要求：
- 字数：1500 字左右
- 结构：引言、正文（3 个论点）、结论
- 风格：专业但易懂
- 包含具体案例
"""
```

### 4.2 代码生成

```python
# 写代码
prompt = """
请用 Python 实现一个函数，功能如下：

函数名：calculate_discount
参数：
- price: 原价（float）
- discount_rate: 折扣率（0-1 之间的 float）
- min_price: 最低价（float，可选，默认 0）

返回值：折后价格（float）

要求：
1. 折扣后价格不能低于 min_price
2. 添加类型注解
3. 添加文档字符串
4. 包含单元测试示例
"""
```

### 4.3 数据分析

```python
# 分析数据
prompt = """
我有以下销售数据：

月份，销售额，利润
1 月，100 万，20 万
2 月，120 万，25 万
3 月，150 万，30 万
4 月，130 万，28 万

请分析：
1. 销售趋势
2. 利润率变化
3. 异常月份及原因推测
4. 改进建议

请用表格和文字说明。
"""
```

### 4.4 学习辅导

```python
# 解释概念
prompt = """
你是一位耐心的老师，擅长用简单的方式解释复杂概念。

请用通俗易懂的方式解释"什么是递归"。

要求：
1. 用生活中的例子类比
2. 包含一个简单的代码示例
3. 说明递归的优缺点
4. 适合零基础学习者理解
"""
```

### 4.5 翻译润色

```python
# 翻译
prompt = """
请将以下技术文档翻译成中文：

[英文原文]

要求：
1. 保持专业术语准确
2. 语句通顺自然
3. 保留代码和专有名词
4. 难懂的地方添加注释
"""

# 润色
prompt = """
请润色以下文字，使其更专业：

[原文]

要求：
1. 修正语法错误
2. 优化表达方式
3. 保持原意不变
4. 适合商务场景
"""
```

## 5. 最佳实践

### 5.1 DO's（推荐做法）

| 做法 | 说明 | 示例 |
|------|------|------|
| **明确具体** | 清晰表达需求 | "写 500 字介绍..." |
| **提供上下文** | 给出背景信息 | "这是一个电商项目..." |
| **给出示例** | 提供参考样例 | "类似这样的格式..." |
| **分步引导** | 引导逐步思考 | "请逐步分析..." |
| **指定格式** | 明确输出格式 | "请用 JSON 格式..." |

### 5.2 DON'Ts（避免做法）

| 做法 | 问题 | 改进 |
|------|------|------|
| **模糊不清** | "写点什么" | "写一篇 500 字的技术文章" |
| **信息不足** | "这个怎么改" | "这段代码有 bug，请帮我修复" |
| **要求过多** | 一次性问 10 个问题 | 分多次提问 |
| **忽略约束** | 不指定格式/字数 | "请用 300 字总结" |

### 5.3 迭代优化

```python
# 第 1 版
prompt_v1 = "介绍人工智能"

# 第 2 版（添加具体要求）
prompt_v2 = "请用 500 字介绍人工智能的定义和发展历程"

# 第 3 版（添加结构和示例）
prompt_v3 = """
请用 500 字介绍人工智能，包括：
1. 定义（100 字）
2. 发展历程（300 字，包括 3 个里程碑）
3. 未来展望（100 字）

风格：通俗易懂，适合非技术人员阅读
"""
```
