## 1. 思维链（Chain of Thought）

### 1.1 基础用法

```
请逐步思考这个问题：
1. 先分析问题
2. 列出已知条件
3. 推导解决方案
4. 给出最终答案
```

### 1.2 效果对比

| 方式 | 准确率 |
|------|--------|
| 直接回答 | 60% |
| 思维链 | 85% |

## 2. 少样本学习（Few-shot Learning）

### 2.1 示例

```
将中文翻译成英文：

中文：你好
英文：Hello

中文：谢谢
英文：Thank you

中文：再见
英文：
```

### 2.2 设计要点

- 示例质量高
- 格式一致
- 覆盖边界情况

## 3. 角色设定（Role Prompting）

```
你是一位资深的 Java 架构师，有 10 年微服务设计经验。
请用专业的角度分析这个系统架构的优缺点...
```

---

## 4. 任务分解（Task Decomposition）

```
请按以下步骤完成任务：

步骤 1：分析需求
步骤 2：设计数据结构
步骤 3：编写核心逻辑
步骤 4：添加错误处理
步骤 5：编写单元测试
```

---

## 5. 自我验证（Self-Verification）

```
请回答这个问题，然后：
1. 检查你的答案是否有逻辑错误
2. 验证是否满足所有约束条件
3. 如有问题，修正你的答案
```

---

## 6. 对比学习（Contrastive Learning）

```
请分析以下两种方案的优缺点：

方案 A：使用 Redis 缓存
方案 B：使用本地缓存

从性能、一致性、成本三个维度对比。
```

---

## 7. 约束条件（Constraints）

```
请写一个 Python 函数，要求：
- 时间复杂度 O(n)
- 空间复杂度 O(1)
- 不使用额外数据结构
- 处理边界情况
```

---

## 8. 格式化输出

```
请用 JSON 格式输出，包含以下字段：
{
  "summary": "...",
  "key_points": [...],
  "recommendations": [...]
}
```

## 9. 高级技巧组合

```
角色：你是一位技术面试官
任务：评估候选人的系统设计能力
步骤：
1. 分析需求
2. 设计架构
3. 识别瓶颈
4. 提出优化方案

请逐步思考，并给出结构化回答。
```
