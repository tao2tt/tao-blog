## 1. 思维链（Chain of Thought）

### 1.1 什么是思维链

**思维链（Chain of Thought，CoT）** 是引导模型逐步推理的技术，通过展示思考过程提高答案准确性。

**核心思想：** 让模型"慢思考"，而不是直接给出答案。

```
传统 Prompt：
"23 × 47 = ?"
→ 直接回答：1081（可能错误）

思维链 Prompt：
"23 × 47 = ?
请逐步计算：
1. 先算 23 × 40
2. 再算 23 × 7
3. 最后相加"
→ 逐步计算，准确率更高
```

### 1.2 基础用法

```python
# 数学问题
prompt = """
小明有 5 个苹果，给了小红 2 个，又买了 3 个，现在有几个？

请逐步思考：
1. 初始有几个苹果？
2. 给了小红后剩几个？
3. 又买了几个？
4. 最后总共几个？

请写出计算过程。
"""

# 逻辑推理
prompt = """
如果所有 A 都是 B，有些 B 是 C，那么以下哪个结论正确？
A. 所有 A 都是 C
B. 有些 A 是 C
C. 有些 A 可能是 C
D. 无法确定

请逐步分析：
1. "所有 A 都是 B"意味着什么？
2. "有些 B 是 C"意味着什么？
3. A 和 C 的关系是什么？
4. 哪个选项正确？
"""
```

### 1.3 效果对比

| 方式 | 数学题准确率 | 逻辑题准确率 |
|------|------------|------------|
| 直接回答 | 60% | 50% |
| 思维链 | 90%+ | 85%+ |

### 1.4 自动思维链

```python
# 让模型自动生成思考过程
prompt = """
请回答以下问题，并在回答前展示你的思考过程：

问题：[你的问题]

思考过程：
"""

# 模型会自动填充思考步骤
```

## 2. 少样本学习（Few-shot Learning）

### 2.1 什么是少样本学习

**少样本学习（Few-shot Learning）** 是通过提供少量示例，让模型学习任务模式。

**原理：** 示例 → 模式识别 → 类比推理

### 2.2 示例设计

```python
# 情感分析示例
prompt = """
判断以下评论的情感倾向（正面/负面/中性）：

评论："这个产品很好用，我很满意"
情感：正面

评论："质量太差了，用了一天就坏了"
情感：负面

评论："还行吧，没什么特别的"
情感：中性

评论："物流很快，但产品一般"
情感：
"""
# 模型会类比输出：中性 或 混合
```

### 2.3 示例数量

| 示例数 | 效果 | 适用场景 |
|--------|------|---------|
| **0 个（Zero-shot）** | 一般 | 简单任务 |
| **1-3 个（Few-shot）** | 好 | 大多数任务 |
| **5+ 个** | 更好 | 复杂任务 |

### 2.4 示例质量

```python
# ❌ 低质量示例
prompt = """
翻译：
你好 → Hello
再见 → Bye
谢谢 → Thanks
"""

# ✅ 高质量示例（多样化、覆盖边界）
prompt = """
翻译（中文→英文）：

正式场合：
"您好，很高兴见到您" → "Hello, nice to meet you"

日常对话：
"吃了吗？" → "Have you eaten?"

商务邮件：
"期待您的回复" → "Looking forward to your reply"

口语表达：
"太棒了！" → "That's awesome!"
"""
```

## 3. 角色设定（Role Prompting）

### 3.1 什么是角色设定

**角色设定** 是让 AI 扮演特定角色，以该角色的视角和专业知识回答问题。

**效果：** 更专业、更有针对性、更符合场景

### 3.2 角色设计

```python
# 技术专家
prompt = """
你是一位资深 Java 架构师，有 10 年微服务设计经验。

请评审以下系统架构设计：
[架构设计内容]

请从以下角度分析：
1. 系统可扩展性
2. 性能瓶颈
3. 容错机制
4. 安全考虑
"""

# 产品经理
prompt = """
你是一位资深产品经理，负责过多个百万用户级产品。

请分析这个产品需求的可行性：
[需求文档]

请从以下角度分析：
1. 用户需求
2. 技术可行性
3. 商业价值
4. 风险评估
"""

# 心理咨询师
prompt = """
你是一位专业的心理咨询师，擅长认知行为疗法。

我最近感到工作压力很大，经常失眠。

请给我一些建议，帮助我缓解压力和改善睡眠。
"""
```

### 3.3 角色 + 任务组合

```python
prompt = """
角色：你是一位资深技术面试官，面试过 500+ 候选人

任务：请面试一位 Java 高级工程师候选人

要求：
1. 提出 5 个技术问题（从基础到深入）
2. 每个问题附带考察点
3. 提供参考答案和评分标准
4. 包含一道编程题
"""
```

## 4. 任务分解（Task Decomposition）

### 4.1 什么是任务分解

**任务分解** 是将复杂任务拆分为多个小步骤，逐步完成。

**好处：**
- ✅ 降低难度
- ✅ 提高准确性
- ✅ 便于调试

### 4.2 分解方法

```python
# 复杂任务
prompt = """
请帮我开发一个用户管理系统。
"""
# ❌ 太复杂，模型难以处理

# 分解后
prompt = """
请帮我设计一个用户管理系统，按以下步骤进行：

步骤 1：需求分析
- 用户角色有哪些？
- 核心功能是什么？

步骤 2：数据库设计
- 需要哪些表？
- 表结构如何设计？

步骤 3：API 设计
- 需要哪些接口？
- 接口定义是什么？

步骤 4：核心代码实现
- 用户注册
- 用户登录
- 用户信息管理

请逐步完成每个步骤。
"""
```

### 4.3 实战案例

```python
# 数据分析任务
prompt = """
请分析这个销售数据集，按以下步骤进行：

步骤 1：数据概览
- 数据量大小
- 字段说明
- 数据质量检查

步骤 2：描述性统计
- 各字段统计信息
- 分布情况

步骤 3：趋势分析
- 月度销售趋势
- 同比增长

步骤 4：洞察与建议
- 关键发现
- 业务建议

数据：
[销售数据]
"""
```

## 5. 自我验证（Self-Verification）

### 5.1 什么是自我验证

**自我验证** 是让模型检查自己的答案，发现并纠正错误。

**方法：**
- ✅ 让模型解释答案
- ✅ 让模型检查逻辑
- ✅ 让模型提供反例

### 5.2 验证技巧

```python
# 方法 1：解释答案
prompt = """
请回答以下问题，然后解释你的答案：

问题：[你的问题]

答案：
解释：
"""

# 方法 2：检查逻辑
prompt = """
请分析以下论证是否有逻辑漏洞：

论证：[论证内容]

请：
1. 总结论证的核心观点
2. 检查前提是否成立
3. 检查推理是否有效
4. 找出可能的逻辑谬误
"""

# 方法 3：提供反例
prompt = """
请判断以下说法是否正确，如果错误请提供反例：

说法："所有鸟都会飞"

请：
1. 判断说法是否正确
2. 如果错误，提供反例
3. 修正说法
"""
```

### 5.3 迭代改进

```python
# 第一轮
prompt_v1 = """
请写一个 Python 函数，计算两个数的最大公约数。
"""

# 第二轮（让模型自我检查）
prompt_v2 = """
以下是计算最大公约数的 Python 函数：

[函数代码]

请检查：
1. 代码是否正确？
2. 有没有边界情况没处理？
3. 有没有更优的实现？
4. 请提供改进版本。
"""
```

## 6. 对比学习（Contrastive Learning）

### 6.1 什么是对比学习

**对比学习** 是让模型对比多个方案/答案，分析优缺点。

**好处：**
- ✅ 更全面
- ✅ 帮助决策
- ✅ 发现盲点

### 6.2 对比技巧

```python
# 方案对比
prompt = """
请对比以下两种技术方案的优缺点：

方案 A：使用 Redis 缓存
方案 B：使用本地缓存

请从以下维度对比：
1. 性能
2. 一致性
3. 成本
4. 复杂度
5. 适用场景

最后给出推荐建议。
"""

# 答案对比
prompt = """
以下是两个不同的答案：

答案 A：[答案 A 内容]
答案 B：[答案 B 内容]

请分析：
1. 两个答案的差异
2. 各自的优缺点
3. 哪个答案更好？为什么？
"""
```

## 7. 约束条件（Constraints）

### 7.1 什么是约束条件

**约束条件** 是明确限制模型输出的范围、格式、内容等。

**作用：**
- ✅ 控制输出质量
- ✅ 符合特定要求
- ✅ 避免无关内容

### 7.2 约束类型

```python
# 字数约束
prompt = """
请用 100 字以内总结这篇文章的核心观点。
"""

# 格式约束
prompt = """
请用 JSON 格式输出，包含以下字段：
{
  "summary": "...",
  "key_points": [...],
  "recommendations": [...]
}
"""

# 内容约束
prompt = """
请介绍人工智能，要求：
1. 只介绍技术层面，不涉及伦理
2. 使用通俗易懂的语言
3. 包含 3 个具体应用案例
4. 避免使用专业术语
"""

# 风格约束
prompt = """
请用正式商务风格写一封邮件。
避免使用口语化表达。
"""
```

### 7.3 多重约束

```python
prompt = """
请写一个 Python 函数，要求：

功能：计算列表的平均值

约束：
1. 函数名：calculate_average
2. 参数：numbers（列表）
3. 返回值：float
4. 添加类型注解
5. 添加文档字符串
6. 处理空列表情况
7. 代码不超过 20 行
8. 包含 3 个单元测试
"""
```

## 9. 高级技巧组合

### 9.1 角色 + 思维链 + 示例

```python
prompt = """
角色：你是一位资深数据科学家

任务：分析以下数据集的异常值

示例：
数据：[1, 2, 3, 100, 5]
异常值：100
原因：远大于其他值

数据：[10, 12, 11, 13, 1000]
异常值：1000
原因：远大于其他值

现在请分析：
数据：[50, 52, 48, 51, 200]

请逐步思考：
1. 计算平均值和标准差
2. 识别偏离较大的值
3. 判断是否为异常值
4. 说明原因
"""
```

### 9.2 任务分解 + 自我验证

```python
prompt = """
请开发一个用户登录系统，按以下步骤进行：

步骤 1：设计数据库表结构
步骤 2：实现登录 API
步骤 3：实现密码加密
步骤 4：实现会话管理

每个步骤完成后，请自我检查：
1. 代码是否正确？
2. 有没有安全漏洞？
3. 有没有边界情况没处理？
4. 如何改进？

最后提供完整的可运行代码。
```
